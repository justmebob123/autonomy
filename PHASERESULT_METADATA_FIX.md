# PhaseResult Metadata Parameter Fix

## Problem
```
error project_planning failed PhaseResult.__init__ got unexpected keyword argument metadata project_planning.py line 124
```

## Root Cause
The `PhaseResult` dataclass uses the parameter name `data`, not `metadata`. The project_planning phase was trying to pass debugging information using `metadata={...}` which caused a TypeError.

## PhaseResult Structure
```python
@dataclass
class PhaseResult:
    success: bool
    phase: str
    task_id: Optional[str] = None
    message: str = ""
    files_created: List[str] = field(default_factory=list)
    files_modified: List[str] = field(default_factory=list)
    errors: List[Dict] = field(default_factory=list)
    data: Dict[str, Any] = field(default_factory=dict)  # ← Use 'data', not 'metadata'
    timestamp: str = ""
```

## Solution
Changed the parameter name from `metadata` to `data` in the error return statement:

**Before:**
```python
return PhaseResult(
    success=False,
    phase=self.phase_name,
    message="Failed to generate expansion plan - no tool calls in response",
    metadata={  # ← Wrong parameter name
        "response_length": len(content) if content else 0,
        "has_content": bool(content),
        "content_preview": content[:200] if content else None
    }
)
```

**After:**
```python
return PhaseResult(
    success=False,
    phase=self.phase_name,
    message="Failed to generate expansion plan - no tool calls in response",
    data={  # ← Correct parameter name
        "response_length": len(content) if content else 0,
        "has_content": bool(content),
        "content_preview": content[:200] if content else None
    }
)
```

## Testing
Verified the fix works correctly:
```python
result = PhaseResult(

    message='test message',
    data={'key': 'value'}
)
# ✓ Works correctly
```
